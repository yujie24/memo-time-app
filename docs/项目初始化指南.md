# 项目初始化指南 - MemoTime App

## 概述

本文档指导你如何将本地初始化的 MemoTime App 项目推送到 GitHub 远程仓库，并配置基本的分支保护和协作规范。

## 前提条件

1. 已安装 Git 并配置全局用户信息（姓名、邮箱）
2. 拥有 GitHub 账户
3. 本地项目已完成初始化（包含 `.gitignore`、`README.md` 等文件）

## 步骤 1：在 GitHub 创建新仓库

### 1.1 登录 GitHub
访问 https://github.com 并登录你的账户。

### 1.2 创建新仓库
1. 点击右上角「+」图标，选择「New repository」
2. 填写仓库信息：
   - **Repository name**：`memo-time-app`（建议使用小写字母和连字符）
   - **Description**：`个人生产力系统 - 集成日历、笔记、健康数据与财务管理的智能应用`
   - **Visibility**：选择 `Public`（公开）或 `Private`（私有）
   - **Initialize this repository with**：
     - ☑️ 不勾选「Add a README file」（本地已有）
     - ☑️ 不勾选「Add .gitignore」（本地已有）
     - ☑️ 不勾选「Choose a license」（可后续添加）
3. 点击「Create repository」按钮

### 1.3 获取远程仓库 URL
创建成功后，复制提供的 HTTPS 或 SSH URL：
- **HTTPS**：`https://github.com/<你的用户名>/memo-time-app.git`
- **SSH**：`git@github.com:<你的用户名>/memo-time-app.git`

推荐使用 HTTPS（无需 SSH 密钥配置），如使用 SSH 请确保已添加公钥到 GitHub。

## 步骤 2：添加远程仓库并推送代码

### 2.1 添加远程仓库
在本地项目根目录（`/app/data/files`）执行：

```bash
git remote add origin https://github.com/<你的用户名>/memo-time-app.git
```

验证远程仓库配置：
```bash
git remote -v
```
应显示：
```
origin  https://github.com/<你的用户名>/memo-time-app.git (fetch)
origin  https://github.com/<你的用户名>/memo-time-app.git (push)
```

### 2.2 推送代码到远程仓库
由于本地分支名为 `main`，使用：

```bash
git push -u origin main
```

参数说明：
- `-u`：设置上游跟踪，后续可直接使用 `git push` 和 `git pull`
- `origin`：远程仓库别名
- `main`：本地分支名

### 2.3 验证推送结果
1. 刷新 GitHub 仓库页面，应显示所有文件
2. 查看提交历史：点击「commits」标签，应看到 "chore: initial commit - project structure and documentation"

## 步骤 3：配置分支保护规则

### 3.1 进入仓库设置
在仓库页面点击「Settings」→「Branches」

### 3.2 添加分支保护规则
1. 点击「Add branch protection rule」
2. 在「Branch name pattern」输入 `main`
3. 配置保护选项（推荐）：
   - ✅ **Require a pull request before merging**
     - ☑️ Require approvals（建议设置为 1）
     - ☑️ Dismiss stale pull request approvals when new commits are pushed
   - ✅ **Require status checks to pass before merging**
     - 可后续配置 CI/CD 工作流后添加具体检查
   - ✅ **Require conversation resolution before merging**
   - ✅ **Require signed commits**（可选，提高安全性）
   - ✅ **Include administrators**（建议勾选，规则对管理员同样生效）
   - ✅ **Do not allow bypassing the above settings**
4. 点击「Create」保存规则

### 3.3 配置协作规范（可选）
在仓库页面点击「Settings」→「Collaboration」
- **Add people**：邀请团队成员
- **Manage access**：设置协作者权限

## 步骤 4：配置 CI/CD 工作流（后续）

### 4.1 创建 GitHub Actions 工作流
在项目根目录创建 `.github/workflows/ci.yml`：

```yaml
name: CI

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.10'
      - name: Install dependencies
        run: |
          python -m pip install --upgrade pip
          pip install -r requirements.txt
      - name: Run tests
        run: |
          pytest
```

### 4.2 添加状态检查到分支保护
CI 工作流运行后，在分支保护规则中添加状态检查要求。

## 步骤 5：后续开发流程

### 5.1 创建功能分支
```bash
git checkout -b feature/your-feature-name
```

### 5.2 提交更改
```bash
git add .
git commit -m "feat: 添加某某功能"
```

### 5.3 推送分支
```bash
git push -u origin feature/your-feature-name
```

### 5.4 创建 Pull Request
1. 访问 GitHub 仓库页面
2. 点击「Compare & pull request」
3. 填写 PR 描述，关联相关 Issue
4. 等待审核和 CI 通过后合并

## 常见问题

### Q1：推送时提示认证失败
**解决方法**：
1. 检查 GitHub 用户名密码（如使用 HTTPS）
2. 或配置 SSH 密钥：
   ```bash
   ssh-keygen -t ed25519 -C "your-email@example.com"
   cat ~/.ssh/id_ed25519.pub
   ```
   将公钥添加到 GitHub Settings → SSH and GPG keys

### Q2：推送时提示分支冲突
**解决方法**：
如果远程仓库已有文件（如初始化的 README），先拉取合并：
```bash
git pull origin main --rebase
git push origin main
```

### Q3：忘记添加某些文件到 .gitignore
**解决方法**：
1. 编辑 `.gitignore` 添加规则
2. 从缓存中移除已跟踪但不应提交的文件：
   ```bash
   git rm --cached 文件名
   ```
3. 重新提交并推送

## 安全建议

1. **敏感信息**：切勿将 `.env`、API 密钥、证书等提交到仓库
2. **依赖安全**：定期更新依赖包，使用 `pip audit` 和 `npm audit` 检查漏洞
3. **代码审查**：所有更改必须通过 Pull Request 审核
4. **分支保护**：启用所有推荐的保护选项
