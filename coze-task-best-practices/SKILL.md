# 扣子任务最佳实践库

本技能库是 PMO Agent 的核心参考手册，用于了解 Worker 能力边界、匹配最佳实践案例，并改写高质量的任务描述。

## 任务目标
- 本 Skill 用于:PMO Agent 需要了解扣子 Agent 能力边界、如何给 Worker 描述任务时
- 能力包含:Worker 能力边界查询、最佳实践案例匹配、高质量任务描述改写
- 触发条件:分派任务前、用户提出模糊需求、需要改写任务描述

## 前置准备
- 依赖说明:本 Skill 无外部依赖，完全通过智能体自然语言能力实现
- 知识准备:熟悉 create_plan_item 工具的 description 字段结构
- 工具检查:如果任务涉及飞书文档、飞书素材等操作，必须先检查用户是否已安装相关飞书工具，如未安装则提醒用户到"我的-工具"中完成安装

## 操作步骤

### 步骤 1:查阅 Worker 能力边界
在分派任务前，先查阅 [references/worker-capability-matrix.md](references/worker-capability-matrix.md)
- 确认 Worker 是否具备所需能力
- 了解核心能力和限制
- 避免分配超出能力边界的任务

### 步骤 2:匹配最佳实践案例
根据用户需求，参考 [references/best-practice-cases.md](references/best-practice-cases.md) 匹配最接近的场景
- 查看对应场景的 Bad 示例和 Good 示例
- 理解 Good 示例的关键改进点
- 提取可复用的改写模式

### 步骤 3:改写任务描述
按照黄金模板改写任务描述：

**黄金模板结构**:
```
<用一句话清晰定义最终要产出什么>

<简述任务背景，或依赖的前序产物路径>

1. <要产出的中间产物 1>
2. <要产出的中间产物 2>
3. <要产出的最终产物，包含风格/格式/长度限制>
```

**核心原则**:描述"产出什么"，而非"怎么做"。重点关注中间产物和最终产物，不指定具体工具名称，不指导 Worker 如何使用工具。

**中间产物 vs 最终产物**:
- **中间产物**: 为达成最终目标需要先产出的内容（如：搜索结果、文案草稿、数据分析、调研报告）
- **最终产物**: 用户最终需要的内容（如：海报图片、播客音频、PPT 文件）
- **关系描述**: 在描述中说明中间产物如何支持最终产物（如"基于搜索结果撰写文案"、"基于文案设计海报"）

**示例说明**:
```
设计一张科技主题海报并归档，需要基于最新 AI 技术趋势。

1. 搜索近 7 天内 AI 技术相关的 5 篇新闻，整理成要点列表
2. 基于搜索结果，撰写海报文案（标题、副标题、标语、关键信息）
3. 基于文案设计科技风格海报，9:16 比例，蓝色主色调
```

### 步骤 4:使用意图增强技巧(可选)
根据任务类型，在【要求】中使用意图增强技巧(见步骤 5)

### 步骤 5:应用意图增强提示
参考 [references/intent-enhancement-tips.md](references/intent-enhancement-tips.md) 增强任务描述:
- 播客增强:脚本风格、角色设定
- 搜索增强:时间范围、来源类型
- 绘图增强:视觉风格、分辨率
- 专家写作增强:报告标准、结构要求
- 代码增强:处理逻辑、异常处理
- 网页增强:单文件 HTML、内联样式和脚本
- 图表增强:图表类型、数据来源、图表与报告联动

### 步骤 6:验证描述质量
确保改写后的描述符合以下标准:
- **目标清晰**: 一句话可概括最终产物
- **背景明确**: 依赖关系和前序产物清晰
- **要求具体**: 中间产物和最终产物明确，产出内容可描述，风格/格式明确
- **输入输出**: 产物保存路径明确
- **避免工具指定**: 不指定具体工具名称、参数、API 调用方式
- **避免操作指导**: 不指导 Worker 如何使用工具或如何执行具体操作

## 注意事项
- 在分派任务前必须先查阅 Worker 能力边界矩阵
- 严禁指定具体工具名称或指导 Worker 如何使用工具
- 重点关注"产出什么"（中间产物和最终产物），而非"怎么做"
- 明确描述中间产物和最终产物的产出内容，而非具体的工具操作步骤
- 对于 PPT 等复杂任务，必须调用专家 Agent，严禁自行生成大纲
- 视觉类任务需注意限制条件（无法生成可编辑源文件、文字渲染能力弱等）
- 播客任务需注意字数与时长关系（1000 字≈3 分钟）
- 飞书工具检查：涉及飞书文档、飞书素材操作时，必须先确认用户已安装相关工具，如未安装需提醒用户到"我的-工具"中完成安装后再继续任务

## 使用示例

### 示例 1:行业调研任务改写
**用户需求**: "去搜一下 AI 的新闻，写个总结放到文档里。"
**参考场景**: 场景 A - 行业调研（含飞书文档）
**改写后**:
```
生成一份关于"Generative AI 在医疗领域应用"的简报并归档。

1. 搜索近 7 天内关于 GenAI 辅助药物研发的 Top 5 新闻，整理成要点列表
2. 基于搜索结果，生成一段 200 字的行业趋势洞察
3. 将搜索结果和洞察整理成简报格式，写入飞书文档
```

### 示例 2:播客制作任务改写
**用户需求**: "把这个文章做成播客。"
**参考场景**: 场景 B - 播客制作
**改写后**:
```
将 `docs/weekly_report.md` 转化为双人对谈风格的播客音频，输入文件为 docs/weekly_report.md

1. 读取输入文档内容，提炼关键信息和观点
2. 基于关键信息，生成双人对谈脚本（Host A: 严肃技术专家 vs Host B: 幽默好奇小白）
3. 基于脚本生成双人对谈风格的播客音频，音色选择"大壹 & 咪仔"
```

### 示例 3:海报设计任务（含中间产物）
**用户需求**: "帮我做个 AI 技术主题的海报。"
**改写后**:
```
设计一张 AI 技术主题海报并归档，需要基于最新 AI 技术趋势。

1. 搜索近 7 天内 AI 技术相关的 5 篇新闻，整理成要点列表
2. 基于搜索结果，撰写海报文案（标题、副标题、标语、关键信息）
3. 基于文案设计科技风格海报，9:16 比例，蓝色主色调
```
