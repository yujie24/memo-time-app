## 播客增强

### 适用场景
需要生成播客音频时，在【要求】中添加脚本风格和角色设定。

### 增强技巧

#### 1. 脚本风格设定
- **轻松幽默**: "脚本风格：轻松幽默的访谈，加入开场白和结束语"
- **严肃专业**: "脚本风格：严谨的技术分析，逻辑清晰，数据支撑"
- **故事叙述**: "脚本风格：以故事为主线，情感充沛，有情节起伏"

#### 2. 角色设定
- **双人对谈**: "Host A: 严肃技术专家 vs Host B: 幽默好奇小白"
- **单人独白**: "风格: 专业播客主播，语速适中，亲和力强"
- **访谈形式**: "Host: 采访者，提出犀利问题；Guest: 行业专家，深度解答"

#### 3. 内容结构
- **开场白**: "加入 30 秒开场白，介绍本期主题和嘉宾"
- **结束语**: "添加 20 秒结束语，总结观点并预告下期"
- **章节划分**: "分为 3 个章节，每个章节 5-8 分钟，有过渡语"

#### 4. 音色选择
- **常见音色**: "大壹"、"咪仔"、"晓雪"、"言言"
- **音色搭配**: "音色选择'大壹 & 咪仔'（大壹稳重，咪仔活泼）"

### 示例
```
1. 读取 `docs/tech_article.md` 内容
2. 生成双人对谈脚本（Host A: 严肃技术专家 vs Host B: 幽默好奇小白）
3. 脚本风格：轻松幽默的访谈，加入开场白和结束语
4. 基于脚本生成音频文件，音色选择"大壹 & 咪仔"
```

## 搜索增强

### 适用场景
需要联网搜索信息时，在【要求】中明确搜索范围和来源类型。

### 增强技巧

#### 1. 时间范围
- **近期**: "搜索范围：近 7 天的新闻"
- **近期扩展**: "搜索范围：近 3 个月的 arXiv 论文"
- **年度回顾**: "搜索范围：2024 年全年的技术趋势报告"

#### 2. 来源类型
- **学术论文**: "搜索来源：arXiv、IEEE、ACM 数字图书馆"
- **科技媒体**: "搜索范围：近 3 个月的 TechCrunch、The Verge 新闻"
- **行业报告**: "搜索来源：Gartner、McKinsey、BCG 的行业研究报告"
- **中文媒体**: "搜索来源：36氪、虎嗅、钛媒体的科技新闻"

#### 3. 搜索关键词
- **精准关键词**: "关键词：Generative AI drug discovery"
- **组合关键词**: "关键词：('AI chip' OR 'neural processing unit') AND 'energy efficiency'"
- **排除关键词**: "关键词：blockchain -cryptocurrency（排除加密货币）"

#### 4. 结果数量
- **精选 Top 5**: "搜索并筛选 Top 5 相关度最高的结果"
- **深度分析 10 篇**: "搜索并阅读 10 篇相关文章进行深度分析"
- **广泛调研 20 篇**: "搜索并收集 20 篇不同观点的文章"

### 示例
```
1. 搜索近 7 天内关于 GenAI 辅助药物研发的 Top 5 新闻（关键词：Generative AI drug discovery）
2. 搜索来源：Nature、Science、MIT Technology Review
3. 生成一段 200 字的行业趋势洞察
```

## 绘图增强

### 适用场景
需要生成图片或视觉设计时，在【要求】中使用结构化描述明确设计需求。

### 增强技巧

#### 1. 结构化描述模板
使用统一的描述格式：**风格 + 比例 + 主体 + 文字内容**

- **完整描述**: "<风格>，<比例>比例。主体：<主体描述>。文字内容：<标题> <小字> <标签>..."
- **简化描述**（无文字）: "<风格>，<比例>比例。主体：<主体描述>"
- **仅设计需求**: "<风格>，<比例>比例，<其他要求>"

#### 2. 风格描述

**常见风格类型**:
- **中国传统风**: "中国传统新年风格"、"中国传统水墨花鸟风格"、"中国风邀请函海报"
- **复古风格**: "复古手账拼贴风"、"复古胶片感"、"复古怀旧风"
- **现代简约**: "极简风格"、"简约标志设计"、"现代简约风"
- **温暖治愈**: "温暖治愈风"、"温馨治愈风格"、"柔和自然"
- **活泼青春**: "活泼青春风格"、"活力青春风"、"年轻时尚"
- **商务专业**: "简洁商务风"、"专业商务风格"、"商务简约"
- **庄重典雅**: "庄重庆典风"、"庄重典雅风格"、"高端大气"
- **校园风格**: "校园风信息图表设计"、"校园清新风"、"学生风格"

**风格组合**:
- 风格 + 用途: "中国传统新年风格海报"、"校园风信息图表设计"
- 风格 + 色调: "温暖治愈风，琥珀光百叶窗光影"
- 风格 + 技法: "中国传统水墨花鸟风格，墨色浓淡渐变"

#### 3. 比例要求

**常见比例**:
- **9:16**: 竖版海报、手机壁纸、视频封面、社交媒体图片
- **3:4**: 竖版海报、信息图表、文档封面
- **1:1**: Logo、图标、社交媒体配图、产品图
- **16:9**: 横版海报、视频封面、PPT 配图
- **21:9**: 超宽横版、电影感封面、公众号封面

**比例描述**:
- "9:16比例"、"3:4比例"、"1:1比例"、"16:9比例"、"21:9"
- "竖版"、"横版"、"方形"、"超宽版"

#### 4. 主体描述

**主体描述要素**:
- **主体物**: "棕色骏马前腿抬起奔腾姿态"、"粘土质感男孩坐在蓝色封面大书上"
- **姿态/动作**: "头部左转鬃毛飘逸"、"坐着书写"、"前腿抬起奔腾"
- **颜色/材质**: "棕色"、"粘土质感"、"米白色宣纸纹理"
- **装饰元素**: "装饰红色花卉纹样头饰及彩色飘带"、"右侧有立体翻页的橙色封面书本"
- **构图/布局**: "前景左侧藤编床头柜及浅棕针织灯罩台灯"、"三个垂直分布元素"
- **背景/环境**: "红绸缎+金色光带"、"暖光室内，琥珀光百叶窗光影"

#### 5. 文字内容描述

**文字内容结构**:
- **标题**: "标题：'龙马精神'"、"标题：'吃遍全城 GO'"
- **副标题**: "副标题：'寻找最地道的美味'"、"小字：'马到成功' '福满家园'"
- **标语/口号**: "标语：'味蕾探险，根本停不下来'"
- **标签/英文**: "标签：'XING YUE TU SHU'"、"标签：'New Year Joy'"
- **多行文字**: "小字：'瑞雪迎门万象新 骏马送福千家乐' '新春大吉' 'New Year Joy'"

**文字排版要求**:
- 字体: "楷体"、"无衬线体"、"手写体"、"香槟色装饰字体"
- 颜色: "米白"、"暖橙"、"金色"、"黑色"
- 字重: "加粗"、"小号"、"大号"
- 位置: "中央"、"左上角"、"右下角"、"顶部"、"底部"

#### 6. 批量生成

**批量生成规则**:
- 使用 `length` 参数指定数量，单次生成多张
- 严禁循环调用或并发调用
- 不同需求的画面可以合并描述，一次生成

**示例**:
```markdown
【要求】:
1. 设计需求: "Logo 设计，1:1比例。风格 1：简约风格，蓝色几何图形。风格 2：商务风格，红金配色。风格 3：科技风格，霓虹灯光效"。
2. 数量要求: 3 张
```

### 示例

2. 数量要求: 1 张
```

#### 示例 4：产品海报（结构化描述）

2. 数量要求: 1 张
```

### 绘图增强最佳实践

#### ✅ 推荐做法
1. **结构化描述**: 使用"风格 + 比例 + 主体 + 文字内容"的统一格式
2. **详细文字**: 文字内容分层描述（标题、副标题、标语、标签）
3. **明确比例**: 指定具体比例（9:16、3:4、1:1 等）
4. **具体主体**: 详细描述姿态、颜色、材质、装饰元素
5. **批量生成**: 使用 length 参数，单次生成多张，避免循环调用
6. **自然语言**: 使用自然语言描述需求，不指定工具参数

#### ❌ 禁忌做法
1. **模糊描述**: 避免风格不明确、主体不具体、文字内容不完整
2. **缺失比例**: 必须指定比例，否则可能不符合使用场景
3. **循环调用**: 批量生成必须使用 length 参数，不可循环调用
4. **指定工具**: 不要指定具体工具名称或参数（如"使用 DALL-E"、"设置 steps=50"）
5. **过度简化**: 描述过于简单，无法生成符合预期的图片

## 专家写作增强

### 适用场景
需要委托专家进行文档写作时，在任务指令中明确报告标准和结构要求。

### 增强技巧

#### 1. 报告标准
- **引用来源**: "引用数据来源，注明参考文献"
- **结构清晰**: "结构清晰，使用金字塔原理，逻辑严密"
- **语气专业**: "语气客观专业，避免主观臆断"

#### 2. 结构要求
- **金字塔原理**: "使用金字塔原理：结论先行，论点支撑，数据论证"
- **章节划分**: "分为 4 个章节：背景分析、问题陈述、解决方案、总结建议"
- **层级明确**: "使用 H2、H3 标题，层级清晰"

#### 3. 内容要求
- **数据支撑**: "每个论点需有数据或案例支撑"
- **案例丰富**: "至少包含 3 个真实案例"
- **对比分析**: "提供 2-3 个方案的对比分析"

#### 4. 格式要求
- **字数限制**: "总字数 2000-3000 字"
- **排版规范**: "使用 Markdown 格式，包含目录、标题、列表、表格"
- **语言风格**: "语言简洁明了，避免冗余表述"

### 示例
```markdown
【要求】:
1. 调用文档专家 Agent。
2. 任务指令: "撰写一份关于 2024 年 AI 技术趋势的行业报告，报告标准：引用数据来源，结构清晰（金字塔原理），语气客观专业。分为背景分析、技术趋势、应用场景、挑战与机遇 4 个章节，总字数 2000-3000 字。"
```

### 网页增强

### 适用场景
需要开发网页时，确保所有代码（HTML 结构、CSS 样式、JavaScript 脚本）都内联在单个 .html 文件中，不拆分成独立的外部文件。

### 增强技巧

#### 1. 代码组织
- **HTML 结构**: "使用语义化标签（header、nav、main、section、footer）构建页面结构"
- **内联样式**: "CSS 样式通过 <style> 标签内联在 <head> 中，包含所有页面样式定义"
- **内嵌脚本**: "JavaScript 代码通过 <script> 标签内嵌在 <body> 底部，包含所有交互逻辑"

#### 2. 依赖处理
- **无外部 CSS/JS**: "不引用外部 .css、.js 文件，所有样式和脚本都在单文件中"
- **CDN 引用**: "如需第三方库（如 jQuery、Bootstrap），使用 CDN 链接在 <head> 中引入"
- **字体引用**: "如需自定义字体，通过 CSS @import 引入或使用系统字体"

#### 3. 交互实现
- **事件处理**: "使用 addEventListener 绑定交互事件，避免 HTML 中的 onclick 等内联事件"
- **动态操作**: "通过 DOM 操作实现页面动态更新和交互效果"
- **表单验证**: "使用 JavaScript 进行表单验证，提供即时反馈"

#### 4. 响应式设计
- **移动端适配**: "使用媒体查询（@media）适配不同屏幕尺寸"
- **弹性布局**: "使用 Flexbox 或 Grid 布局，确保页面响应式显示"
- **视口设置**: "在 <head> 中添加 <meta name='viewport' content='width=device-width, initial-scale=1.0'>"

#### 5. 代码规范
- **格式规范**: "HTML、CSS、JavaScript 代码格式规范，添加必要注释"
- **命名规范**: "类名、ID、变量名使用驼峰或短横线命名，语义清晰"
- **单文件输出**: "确保所有代码都在单个 .html 文件中，可直接在浏览器打开"

### 示例

#### 示例 1：企业官网首页
```
生成一个企业官网首页，单文件 HTML 格式。

1. 页面结构: 使用语义化标签构建（header、nav、hero section、features section、footer）
2. 内联样式: 所有 CSS 样式通过 <style> 标签写在 <head> 中，包含响应式设计（使用 @media 适配移动端）
3. 内嵌脚本: JavaScript 代码通过 <script> 标签写在 <body> 底部，实现导航栏滚动效果和表单提交验证
4. 要求: 不引用外部 .css、.js 文件，所有代码在单个 .html 文件中，可直接在浏览器打开
```

#### 示例 2：数据可视化仪表板
```
生成一个数据可视化仪表板页面，单文件 HTML 格式。

1. 页面结构: 包含标题栏、数据卡片区、图表展示区、筛选工具栏
2. 内联样式: 所有 CSS 样式通过 <style> 标签写在 <head> 中，使用 Flexbox 布局实现响应式设计
3. 内嵌脚本: JavaScript 代码通过 <script> 标签写在 <body> 底部，实现数据加载、图表渲染和交互逻辑
4. 第三方库: 如需图表库（如 Chart.js），使用 CDN 链接在 <head> 中引入
5. 要求: 不引用外部 .css、.js 文件（第三方库除外），所有自定义代码在单个 .html 文件中
```

#### 示例 3：交互式表单页面
```
生成一个用户注册表单页面，单文件 HTML 格式。

1. 页面结构: 包含表单标题、输入字段（用户名、邮箱、密码、确认密码）、提交按钮、提示信息区
2. 内联样式: 所有 CSS 样式通过 <style> 标签写在 <head> 中，包含表单样式、错误提示样式、成功提示样式
3. 内嵌脚本: JavaScript 代码通过 <script> 标签写在 <body> 底部，实现实时表单验证、密码强度检测、提交处理逻辑
4. 交互要求: 使用 addEventListener 绑定事件，避免内联 onclick；提供即时验证反馈（红框提示错误、绿框提示正确）
5. 要求: 不引用外部 .css、.js 文件，所有代码在单个 .html 文件中，可直接在浏览器打开
```

#### ❌ 禁忌做法
1. **外部文件引用**: 不要创建独立的 .css、.js 文件并在 HTML 中通过 <link> 或 <script src> 引用
2. **内联事件**: 避免使用 HTML 中的 onclick、onchange 等内联事件处理器
3. **硬编码数据**: 避免将数据硬编码在 JavaScript 中，应通过 API 获取或配置对象管理
4. **缺少响应式**: 必须包含媒体查询，适配移动端和桌面端
5. **无注释**: 代码应添加必要的注释，提高可维护性

## 图表增强

### 适用场景
需要在报告中添加数据可视化图表或流程图时，在任务描述中明确图表类型、数据来源和展示目的。

### 增强技巧

#### 1. 图表类型明确（ECharts）
- **折线图**: "生成折线图，展示时间序列趋势（如月度销售额、用户增长）"
- **柱状图**: "生成柱状图，展示类别对比（如各产品销售额、各渠道转化率）"
- **饼图**: "生成饼图，展示占比分布（如用户性别比例、市场份额）"
- **面积图**: "生成面积图，展示累积趋势（如累计用户数、累计销售额）"
- **词云**: "生成词云图，展示关键词频率（如用户评论高频词、搜索热词）"
- **混合图表**: "生成混合图表，在同一图表中展示折线趋势和柱状对比"

#### 2. 架构图类型明确（Draw.io）
- **流程图**: "生成流程图，展示业务流程（如用户注册流程、订单处理流程）"
- **思维导图**: "生成思维导图，展示知识结构（如技术栈体系、功能模块划分）"
- **架构图**: "生成架构图，展示系统架构（如微服务架构、应用架构、部署架构）"
- **组织架构图**: "生成组织架构图，展示团队结构（如部门组织、汇报关系）"
- **甘特图**: "生成甘特图，展示项目计划（如开发时间线、里程碑节点）"
- **时间轴**: "生成时间轴，展示发展历程（如产品演进、公司发展）"

#### 3. 图表内容要求
- **数据来源**: "图表数据来源：输入文件 `data/sales.csv` 或分析结果"
- **展示目的**: "图表用于展示月度销售额趋势，辅助分析季节性特征"
- **数据维度**: "X轴：月份（1-12月），Y轴：销售额（万元），系列：2023年、2024年对比"
- **样式要求**: "图表样式：蓝色系配色，包含数据标签、图例、标题"

#### 4. 图表与报告联动
- **中间产物**: "先生成数据可视化图表，再基于图表撰写分析报告"
- **图表引用**: "报告中包含对图表的解读，说明数据趋势和关键发现"
- **章节对应**: "报告每个章节对应一个图表，图文结合增强可读性"

#### 5. 多图表组合
- **图表数量**: "生成 3 个图表：折线图（趋势）、饼图（占比）、柱状图（对比）"
- **图表顺序**: "图表按报告章节顺序排列：趋势分析 -> 类别分析 -> 产品分析"
- **统一风格**: "所有图表使用统一的配色方案和样式风格"

### 示例

#### 示例 1：销售分析报告 + ECharts 图表
```
生成一份销售数据分析报告，包含数据可视化图表。

1. 读取 `data/sales_2024.csv` 数据文件，按月份统计销售额
2. 生成折线图：X轴为月份（1-12月），Y轴为销售额（万元），展示全年销售趋势
3. 基于折线图，撰写分析报告，包含以下章节：
   - 数据概览（总销售额、同比增长）
   - 趋势分析（解读折线图趋势，识别高峰和低谷）
   - 季节性特征（分析月度波动规律）
   - 结论与建议（数据驱动建议）
```

#### 示例 2：技术方案文档 + Draw.io 架构图
```
生成一份微服务架构技术方案，包含架构图。

1. 明确技术架构：Spring Boot + MySQL + Redis + RabbitMQ
2. 生成架构图，包含以下层次：
   - 客户端层：Web端、移动端
   - 网关层：API Gateway
   - 服务层：用户服务、商品服务、订单服务
   - 数据层：MySQL、Redis
   - 消息层：RabbitMQ
3. 基于架构图，撰写技术文档，包含以下章节：
   - 架构概述（整体设计原则）
   - 服务设计（各服务职责）
   - 技术选型（各组件选型理由）
   - 数据设计（数据库分库分表策略）
```

#### 示例 3：产品规划 + Draw.io 甘特图
```
生成一份产品开发计划，包含甘特图。

1. 明确开发周期：2024年3月至2024年8月
2. 生成甘特图，包含以下阶段：
   - 需求阶段（3月）：需求调研、需求分析、原型设计
   - 设计阶段（4月）：UI设计、交互设计
   - 开发阶段（5-6月）：前端开发、后端开发
   - 测试阶段（7月）：功能测试、Bug修复
   - 上线阶段（8月）：灰度发布、正式上线
   - 关键里程碑：需求冻结、设计定稿、开发完成、正式上线
3. 基于甘特图，撰写开发计划文档，包含以下章节：
   - 项目概述（产品背景、目标用户）
   - 时间规划（解读甘特图，说明各阶段任务）
   - 资源规划（人员配置）
   - 风险管理（潜在风险、应对措施）
```

#### ❌ 禁忌做法
1. **指定工具**: 不要指定具体工具名称（如"使用 ECharts"、"使用 Draw.io"）
2. **模糊图表类型**: 必须明确图表类型（折线图、柱状图、饼图、流程图、架构图等）
3. **缺少数据来源**: 必须说明图表数据来源（输入文件、分析结果）
4. **图表与报告分离**: 图表应作为报告的中间产物，与报告内容紧密结合
5. **图表数量不清**: 必须明确生成几个图表，每个图表的类型和用途

## 意图增强最佳实践

### 1. 选择性使用
- 根据任务类型选择相应的增强技巧
- 不要过度使用，保持描述简洁
- 优先关注"必须明确"的信息

### 2. 自然语言表达
- 使用自然语言描述需求
- 避免使用技术术语或工具参数
- 让智能体理解"要什么"，而非"怎么做"

### 3. 量化与具体化
- 尽量使用量化指标（字数、数量、时间）
- 提供具体的范围和限制
- 明确输出格式和路径

### 4. 一致性
- 在同一项目中保持风格一致
- 术语使用前后统一
- 格式规范统一（如日期格式、路径格式）

### 5. 可验证性
- 描述应可被验证是否完成
- 提供明确的验收标准
- 输出结果应可检查和验证
