## 场景 A：行业调研（含飞书文档）

### 飞书工具检查
在使用此场景前，必须先确认用户是否已安装飞书文档相关工具，如未安装需提醒用户到"我的-工具"中完成安装后再继续任务。

### 用户原始需求
"去搜一下 AI 的新闻，写个总结放到文档里。"

### ❌ Bad (拒收)
范围太广，未指定搜索源，未指定具体的飞书文档位置。

### ✅ Good (改写参考)
```
生成一份关于"Generative AI 在医疗领域应用"的简报并归档。

1. 搜索近 7 天内关于 GenAI 辅助药物研发的 Top 5 新闻（关键词：Generative AI drug discovery）
2. 生成一段 200 字的行业趋势洞察
3. 将内容写入飞书文档
```

### 改写要点
- 明确搜索关键词和时间范围（"近 7 天内"）
- 指定具体的主题领域（"GenAI 辅助药物研发"）
- 量化输出要求（"Top 5 新闻"、"200 字"）

## 场景 B：播客制作

### 用户原始需求
"把这个文章做成播客。"

### ❌ Bad (拒收)
指定具体工具名称，或未指定音色/模式。

### ✅ Good (改写参考)
```
将 `docs/weekly_report.md` 转化为双人对谈风格的播客音频，输入文件为 docs/weekly_report.md

1. 读取 `docs/weekly_report.md` 内容
2. 生成一个双人对谈脚本（Host A: 严肃技术专家 vs Host B: 幽默好奇小白）
3. 基于脚本生成音频文件，音色选择"大壹 & 咪仔"
```

### 改写要点
- 明确输入文件路径
- 指定脚本风格（"双人对谈"、"严肃 vs 幽默"）
- 明确音色选择（"大壹 & 咪仔"）
- 未指定具体工具名称，只描述产出

## 场景 C：专家 PPT 制作

### 用户原始需求
"帮我做个 PPT。"

### ❌ Bad (拒收)
自行生成大纲，或尝试指导专家如何排版。

### ✅ Good (改写参考)
```
委托 PPT 专家制作一份关于"2024 营销规划"的演示文稿。

1. 调用 PPT 专家 Agent
2. 任务指令: "制作一份 2024 营销规划 PPT，包含市场分析、策略制定、预算分配三部分，风格简洁商务"
3. 注意：不要自行生成大纲，直接将任务指令透传给专家
```

### 改写要点
- 明确调用专家 Agent
- 直接透传任务指令，不自行生成内容
- 任务指令包含：主题、内容框架、风格要求
- 强调"不要自行生成大纲"
- 输出为专家返回的结果

## 场景 D：视觉设计

### D1：海报设计（含文字）

#### 用户原始需求
"帮我设计一张中国风的新年海报。"

#### ❌ Bad (拒收)
未指定风格细节、比例、主体、文字内容，描述过于简单。

#### ✅ Good (改写参考)
```
生成中国传统新年风格海报。

1. 设计需求: "中国传统新年风格海报，9:16比例。主体：棕色骏马前腿抬起奔腾姿态，头部左转鬃毛飘逸，装饰红色花卉纹样头饰及彩色飘带。文字内容：标题'龙马精神'，小字'马到成功' '福满家园' '瑞雪迎门万象新 骏马送福千家乐' '新春大吉' 'New Year Joy' '马蹄声声远 春风送暖来 佳节人团圆 幸福乐开怀' '2026'"
2. 数量要求: 1 张
```

#### 改写要点
- 使用结构化描述：风格 + 比例 + 主体 + 文字内容
- 文字内容详细：标题、小字、标签分层描述
- 比例明确：9:16、3:4、1:1 等
- 主体描述具体：姿态、颜色、装饰元素
- 批量生成使用 length 参数，避免循环调用

## 场景 F：数据分析报告 + ECharts 可视化

### 用户原始需求
"帮我写个销售数据分析报告，再做个图表。"

### ❌ Bad (拒收)
未指定分析维度、数据来源、图表类型，未明确报告结构。

### ✅ Good (改写参考)
```
生成一份 2024 年度销售数据分析报告，包含数据可视化图表。

1. 读取 `data/sales_2024.csv` 数据文件
2. 进行数据分析：
   - 按月份统计销售额趋势
   - 按产品类别统计销售额占比
   - 识别 Top 5 最佳销售产品
   - 计算同比增长率
3. 生成数据可视化图表：
   - 折线图：展示月度销售额趋势（1-12月）
   - 饼图：展示各产品类别销售额占比
   - 柱状图：展示 Top 5 产品销售额对比
4. 基于分析结果和图表，撰写数据报告，包含以下章节：
   - 数据概览（总销售额、同比增长、关键发现）
   - 趋势分析（月度趋势解读、季节性特征）
   - 类别分析（各类别表现、机会点）
   - 产品分析（Top 产品分析、改进建议）
   - 结论与建议（数据驱动建议）
```

### 改写要点
- 明确输入数据文件路径
- 具体化分析维度（按月份、按类别、Top 产品）
- 明确图表类型和展示内容（折线图、饼图、柱状图）
- 报告结构清晰，包含数据概览、趋势分析、类别分析、产品分析、结论建议
- 中间产物（数据分析、可视化图表）明确，用于支持最终报告撰写
- 不指定具体工具名称，只描述要产出的图表类型和内容

## 改写原则总结

### ✅ 推荐做法
1. **产出导向**: 描述"产出什么"（中间产物和最终产物），而非"怎么做"
2. **中间产物明确**: 明确描述需要先产出的中间内容（如搜索结果、文案草稿、数据分析）
3. **最终产物清晰**: 清晰描述最终要产出的内容（如海报图片、播客音频、PPT 文件）
4. **关系明确**: 说明中间产物如何支持最终产物（如"基于搜索结果撰写文案"、"基于文案设计海报"）
5. **具体量化**: 使用明确的时间、数量、字数等量化指标
6. **路径明确**: 提供具体的输入文件路径和输出目录路径
7. **风格清晰**: 明确风格、格式、语气等要求
8. **调用专家**: 复杂任务（如 PPT）直接调用专家，不自行生成内容
9. **自然语言**: 使用自然语言描述需求，不指定工具参数

### ❌ 禁忌做法
1. **指定工具**: 不要指定具体工具名称（如"使用 DALL-E 生成"）
2. **指导操作**: 不要指导 Worker 如何使用工具
3. **操作步骤**: 不要描述具体的操作步骤（如"打开文件"、"选择工具"），而是描述要产出什么
4. **循环调用**: 批量任务必须一次性生成，不可循环调用
5. **自行生成**: PPT 等复杂任务严禁自行生成大纲
6. **模糊需求**: 避免范围太广、无时间限制、无具体主题的需求
7. **缺失中间产物**: 对于需要多步处理的任务，必须明确描述中间产物

## 案例匹配技巧

当收到用户需求时，按照以下步骤匹配案例：

1. **识别任务类型**: 搜索/创作/设计/数据处理
2. **查看对应场景**: 找到最匹配的场景示例
3. **对比 Bad/Good**: 理解 Bad 示例的问题和 Good 示例的改进点
4. **提取模板**: 提取可复用的模板结构
5. **填充内容**: 根据用户具体需求填充模板内容
