## 2. HealthKit 集成方案

### 2.1 支持的数据类型

| 数据类型 | HealthKit 标识符 | 单位 | 实时监控 | 敏感级别 |
|----------|------------------|------|----------|----------|
| **步数** | `HKQuantityTypeIdentifier.stepCount` | 步 | 是 | 低 |
| **心率** | `HKQuantityTypeIdentifier.heartRate` | 次/分钟 | 是 | 高 |
| **睡眠分析** | `HKCategoryTypeIdentifier.sleepAnalysis` | 小时 | 是 | 中 |
| **体重** | `HKQuantityTypeIdentifier.bodyMass` | kg | 否 | 高 |
| **活动能量** | `HKQuantityTypeIdentifier.activeEnergyBurned` | 千卡 | 否 | 低 |
| **步行距离** | `HKQuantityTypeIdentifier.distanceWalkingRunning` | 公里 | 否 | 低 |

### 2.2 权限管理流程

```swift
// 1. 检查设备支持
HKHealthStore.isHealthDataAvailable()

// 2. 请求权限
healthStore.requestAuthorization(
    toShare: writeTypes,
    read: readTypes
)

// 3. 检查授权状态
healthStore.authorizationStatus(for: sampleType)
```

**权限申请配置** (Info.plist):
```xml
<key>NSHealthShareUsageDescription</key>
<string>MemoTime需要读取您的健康数据（步数、心率、睡眠等），以便在个人数据面板中提供健康状态概览和趋势分析</string>

<key>NSHealthUpdateUsageDescription</key>
<string>MemoTime需要保存健康数据分析结果，以便为您提供个性化健康建议</string>
```

### 2.3 数据查询接口

#### 当日汇总数据
```swift
func queryTodaySummary(for dataType: HealthDataType) async throws -> Double
```
- **实现**: 使用 `HKStatisticsQuery` 计算指定类型的累计总和
- **示例**: 查询今日步数总和

#### 时间段详细数据
```swift
func queryHealthData(for dataType: HealthDataType,
                    from startDate: Date,
                    to endDate: Date) async throws -> [HealthDataSample]
```
- **实现**: 使用 `HKSampleQuery` 获取指定时间范围的数据样本
- **应用**: 生成趋势图表、详细分析

#### 周度趋势数据
```swift
func queryWeeklyTrend(for dataType: HealthDataType) async throws -> [Date: Double]
```
- **实现**: 查询过去7天数据，按日期分组计算每日平均值
- **输出**: 用于趋势分析和可视化

## 3. 实时同步机制设计

### 3.1 架构概览

```
健康数据变化 → HealthKit → ObserverQuery 监听 → 增量数据获取 → 本地加密存储 → 云同步（可选）
```

### 3.2 核心组件

#### 1. Observer Query（实时监控）
```swift
let query = HKObserverQuery(sampleType: sampleType, predicate: nil) { 
    query, completionHandler, error in
    // 1. 错误处理
    // 2. 获取最新数据
    // 3. 更新本地存储
    // 4. 触发云同步
}
```

#### 2. Anchored Object Query（增量同步）
```swift
let query = HKAnchoredObjectQuery(
    type: sampleType,
    predicate: nil,
    anchor: lastAnchor,  // 上次同步锚点
    limit: HKObjectQueryNoLimit
) { query, newSamples, deletedSamples, newAnchor, error in
    // 1. 处理新增数据
    // 2. 处理删除数据  
    // 3. 更新锚点
}
```

### 3.3 同步频率策略

| 数据类型 | 监控频率 | 后台交付 | 触发场景 |
|----------|----------|----------|----------|
| **步数** | 每小时 | 启用 | 达到运动目标、长时间静坐 |
| **心率** | 实时 | 启用 | 心率异常、运动状态变化 |
| **睡眠** | 每天 | 启用 | 睡眠开始/结束、睡眠阶段变化 |
| **体重** | 手动 | 禁用 | 用户手动录入、第三方设备同步 |

### 3.4 数据去重与合并

#### 去重策略
- **基于 UUID**: 使用 HealthKit 样本的唯一标识符
- **基于时间戳**: 相同类型、相同时间的数据去重
- **基于来源**: 区分不同设备来源的数据

#### 合并逻辑
```swift
struct DataMergeLogic {
    // 1. 按时间窗口合并（如：5分钟内的心率数据取平均）
    // 2. 按数据源优先级排序（Apple Watch > iPhone > 第三方设备）
    // 3. 处理冲突数据（时间戳相同但数值不同）
}
```

## 4. 隐私保护实现

### 4.1 加密架构

#### 三层加密体系
```
Layer 1: 用户主密钥（PBKDF2-SHA256, 100k iterations）
        ↓
Layer 2: 数据类型密钥（HKDF-SHA256）
        ↓  
Layer 3: 记录级别加密（AES-256-GCM）
```

#### 密钥派生流程
```swift
// 1. 用户密码 + 设备盐值 → 主密钥
let masterKey = deriveKey(password: userPassword, 
                         salt: deviceSalt,
                         algorithm: .pbkdf2SHA256)

// 2. 主密钥 + 数据类型 → 数据类型密钥
let dataTypeKey = deriveDataTypeKey(masterKey: masterKey,
                                   dataType: .heartRate)

// 3. 数据类型密钥 + 随机 nonce → 记录加密
let encryptedData = AES.GCM.seal(data, using: dataTypeKey, nonce: nonce)
```

### 4.2 本地存储安全

#### 存储结构
```
📁 MemoTime/
 └── 📁 HealthData/
     └── 📁 Encrypted/
         ├── sample1_uuid_timestamp.enc
         ├── sample2_uuid_timestamp.enc
         └── .stats.json
```

#### 文件保护
```swift
// 文件系统级保护
let attributes: [FileAttributeKey: Any] = [
    .protectionKey: FileProtectionType.complete
]
```

### 4.3 云同步安全

#### 传输协议
```
设备端 → AES-256-GCM 加密 → TLS 1.3 传输 → 云服务 → 目标设备 → 解密验证
```

#### 密钥管理
- **用户主密钥**: 仅本地存储，永不传输
- **会话密钥**: ECDH 临时密钥协商，前向保密
- **信封加密**: 加密密钥的加密，保护传输安全

#### 隐私控制选项
| 选项 | 默认状态 | 用户控制 | 影响范围 |
|------|----------|----------|----------|
| **本地加密存储** | 启用 | 不可关闭 | 所有数据 |
| **云同步** | 禁用 | 用户选择 | 可配置 |
| **数据类型监控** | 步数、心率、睡眠 | 用户选择 | 按需启用 |
| **数据保留期限** | 3个月 | 用户选择 | 自动清理 |
