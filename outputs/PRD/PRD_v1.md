## 1. 系统概述

### 1.1 产品愿景
MemoTime App 是一款面向个人用户的集成生产力系统，深度整合日历、笔记、日记、项目管理功能，并通过智能自动化引擎实现链接内容的自动分类与多格式保存。

### 1.2 核心价值主张
- **智能化**：自动识别链接内容，智能分类到对应项目、笔记或任务
- **一体化**：统一管理日程、笔记、日记和项目，消除工具切换成本
- **自动化**：从链接分享到内容保存、分类、索引的全流程自动化
- **隐私优先**：本地处理优先，端到端加密，用户数据完全可控

### 1.3 目标用户
- 知识工作者、研究人员、学生等需要大量收集和整理网络信息的用户
- 追求高效率、喜欢自动化工作流的科技爱好者
- 重视数据隐私，希望数据存储在本地或可控云端的用户

### 1.4 差异化竞争优势（基于竞品分析结论）
1. **混合智能分类系统**：结合规则引擎与AI模型，提供高准确率且可解释的分类结果
2. **跨平台无缝体验**：iOS/Mac原生深度集成 + Web端完整功能，数据实时同步
3. **隐私优先架构**：本地处理优先，端到端加密，提供可选云同步但非强制
4. **开发者友好生态**：开放API设计，支持插件扩展，构建可持续的社区生态

## 2. 核心功能模块描述（基于竞品分析架构建议）

### 2.1 多源链接抓取模块
**参考竞品**：Obsidian LinkStowr 的 Chrome 扩展思路
**核心功能**：
- **浏览器扩展**：支持 Chrome/Safari/Firefox，捕获用户分享的链接
- **移动端分享集成**：支持 iOS 分享菜单、Android 分享面板
- **API 接收端点**：提供 RESTful API 接收外部应用推送的链接
- **链接去重机制**：基于 MD5 或相似度算法避免重复内容

**技术实现**：
- 前端：JavaScript 扩展监听页面操作
- 后端：接收队列处理，支持批量异步处理
- 数据验证：URL 有效性检查、安全性过滤

### 2.2 智能分类引擎
**参考竞品**：融合 Roam Research 图关联算法 + Apple Notes 语义理解
**核心功能**：
- **双层分类系统**：
  - 第一层：基于规则的关键词匹配（快速、可解释）
  - 第二层：基于 BERT 微调模型的语义理解（精准、智能）
- **分类标签体系**：
  - 项目关联：自动关联到现有项目
  - 主题分类：技术、设计、商业、生活等
  - 内容类型：文章、教程、论文、新闻、视频等
  - 优先级：重要、一般、参考
- **用户反馈闭环**：分类结果可手动调整，系统学习用户偏好

**技术实现**：
- 规则引擎：正则表达式 + 关键词库
- AI 模型：基于 Hugging Face Transformers 的 BERT 微调
- 图数据库：Neo4j 或 Dgraph 存储实体关系

### 2.3 格式转换管道
**参考竞品**：参考 Microsoft Graph API 的多格式支持
**核心功能**：
- **多格式输出**：支持 PDF、PPTX、DOCX 三种主要格式
- **智能格式选择**：根据内容类型自动推荐最佳格式
- **模板系统**：支持用户自定义输出模板
- **批量转换**：支持多个链接的批量格式转换

**技术实现**：
- PDF 生成：Puppeteer + HTML 转 PDF（高保真）
- PPT 导出：基于模板的 JSON 到 PPTX 转换
- Word 保存：Markdown 到 DOCX 标准化转换
- 异步处理：Celery 或类似任务队列

### 2.4 API 设计原则
**参考竞品**：采用 Notion 的 RESTful API 设计
**核心功能**：
- **完整的 RESTful API**：符合 OpenAPI 规范
- **OAuth 2.0 认证**：支持第三方应用集成
- **Webhook 支持**：实时通知链接处理状态
- **开发者文档**：完整的 API 文档和 SDK

## 3. 链接处理详细流程（基于技术选型方向）

### 3.1 全流程概览
```
用户分享链接 → 链接捕获 → 内容抓取 → 智能分类 → 格式转换 → 索引存储 → 用户通知
```

### 3.2 详细步骤说明

#### 步骤1：链接捕获（前端）
- **触发方式**：
  1. 浏览器扩展：用户点击"保存到 MemoTime"按钮
  2. 移动端：分享菜单选择 MemoTime
  3. API 调用：其他应用通过 API 推送链接
- **数据收集**：
  - 原始 URL
  - 页面标题（如可用）
  - 用户添加的备注标签
  - 目标分类建议（可选）

#### 步骤2：内容抓取（后端）
- **技术栈**：Puppeteer（网页渲染）+ Readability（内容提取）
- **处理流程**：
  1. URL 验证与安全性检查
  2. 使用 Puppeteer 渲染页面（支持 JavaScript 动态内容）
  3. 应用 Readability 算法提取核心内容
  4. 提取元数据：标题、作者、发布时间、关键词
  5. 提取多媒体：图片、表格、代码片段
- **容错机制**：
  - 超时处理：默认 30 秒超时
  - 重试策略：3 次重试，指数退避
  - 降级方案：静态 HTML 解析作为备选

#### 步骤3：智能分类
- **第一层：规则引擎分类**
  - 关键词匹配：预设分类规则库
  - 正则表达式：识别特定模式（如 GitHub 仓库、学术论文）
  - 优先级：高优先级规则优先匹配
- **第二层：AI 模型分类**
  - 输入：页面标题 + 提取的内容摘要
  - 模型：基于 BERT 的多标签分类模型
  - 输出：分类标签 + 置信度分数
- **决策融合**：
  - 规则引擎结果置信度 > 0.9 → 直接采用
  - AI 模型置信度 > 0.8 → 采用 AI 结果
  - 两者冲突 → 人工标注队列或用户选择

#### 步骤4：格式转换
- **格式选择逻辑**：
  - **PDF**：长文章、学术论文、需要保真打印的文档
  - **PPTX**：演示文稿、教程、带有大量图片的内容
  - **DOCX**：需要编辑的文档、报告、笔记整理
- **转换流程**：
  1. 清洗与标准化：去除广告、无关元素
  2. 应用样式模板：根据用户偏好或内容类型
  3. 生成目标格式文件
  4. 质量检查：文件完整性、格式正确性

#### 步骤5：索引存储
- **结构化存储**（PostgreSQL）：
  - 链接元数据表
  - 分类标签表
  - 用户项目关联表
  - 处理状态表
- **文档存储**（对象存储/S3 兼容）：
  - 原始 HTML 备份
  - 提取的纯文本
  - 生成的 PDF/PPTX/DOCX 文件
- **全文搜索索引**（Elasticsearch）：
  - 标题、摘要、全文内容索引
  - 支持标签、项目、时间范围过滤

#### 步骤6：用户通知与集成
- **实时通知**：处理完成推送到用户设备
- **日历集成**：带有时间敏感性的内容添加到日历
- **笔记关联**：自动关联到相关笔记页面
- **项目更新**：更新对应项目的参考资料库

## 4. 用户故事（至少5个主要场景）

### 用户故事1：学术研究人员的工作流
**角色**：大学研究员，需要收集大量学术论文和参考资料
**场景**：在浏览 arXiv、Google Scholar 时发现相关论文
**需求**：
- 一键保存论文链接，自动提取 PDF 版本
- 智能分类到对应研究项目
- 自动提取论文摘要、关键词、参考文献
- 与项目进度关联，提醒阅读截止时间
**验收标准**：
- 论文保存成功率 > 95%
- 自动分类准确率 > 85%
- PDF 版本获取成功率 > 90%

### 用户故事2：产品经理的需求收集
**角色**：互联网产品经理，需要收集竞品分析和用户反馈
**场景**：浏览竞品网站、用户社区、产品评测文章
**需求**：
- 保存竞品功能截图和描述
- 自动分类到对应竞品分析项目
- 提取关键数据点（价格、功能、用户评价）

**验收标准**：
- 页面内容提取完整率 > 90%
- 竞品信息结构化提取准确率 > 80%
- PPT 报告生成时间 < 3分钟

### 用户故事3：学生的课程资料整理
**角色**：大学生，需要整理各门课程的在线资料
**场景**：老师分享的课件链接、参考阅读文章、视频教程
**需求**：
- 按课程自动分类保存的资料
- 支持多种格式（PDF 课件、Word 文档、视频链接）

**验收标准**：
- 课程分类准确率 > 90%

**场景**：浏览新闻、社交媒体、其他创作者作品

## 6. 非功能需求

### 6.1 性能需求
- **系统响应时间**：API响应时间 < 200ms（P95）
- **链接处理时间**：从捕获到可用 < 60秒（P95）
- **并发处理能力**：支持同时处理 100+ 链接

### 6.2 安全需求
- **数据加密**：传输层 TLS 1.3，存储数据端到端加密
- **访问控制**：基于角色的细粒度权限控制
- **隐私保护**：本地处理优先，用户数据所有权明确
- **安全审计**：完整的操作日志和访问记录

### 6.3 可用性需求
- **系统可用性**：99.9% 正常运行时间（SLA）
- **跨平台支持**：iOS 16+、macOS 13+、主流浏览器
- **离线可用性**：核心功能支持离线操作
- **可访问性**：符合 WCAG 2.1 AA 标准

### 6.4 可维护性需求
- **监控告警**：完整的系统监控和异常告警
- **文档完整性**：技术文档、API文档、用户手册
