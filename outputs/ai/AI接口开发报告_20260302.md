# MemoTime App AI接口开发报告

## 1. 项目概述

基于冲刺第二周计划，已完成AI基础接口简化开发。本报告记录了AI分析接口的设计、实现和测试结果。

### 1.1 开发目标
- 实现Coze API集成（实际采用DeepSeek API作为示例）
- 开发数据预处理模块，从各模块收集最近7天数据
- 创建分析结果展示界面
- 完成基础功能验证

### 1.2 技术方案
采用模块化设计，分为以下核心组件：
1. **DeepSeek API客户端** - 处理与AI服务商的通信
2. **数据收集器** - 从财务、链接、日历、健康模块收集数据
3. **分析服务** - 整合数据收集和AI分析，提供完整服务
4. **iOS端管理器和界面** - 移动端集成

## 2. 系统架构

### 2.1 组件关系
```
┌─────────────────────────────────────────────┐
│                AnalysisService             │
│  ┌─────────────┐    ┌──────────────────┐  │
│  │DataCollector│    │ DeepSeekClient   │  │
│  └─────────────┘    └──────────────────┘  │
└─────────────────────────────────────────────┘
        │                          │
        ▼                          ▼
┌──────────────────┐    ┌──────────────────┐
│   各模块数据库   │    │  DeepSeek API    │
│ (financial.db等) │    │                  │
└──────────────────┘    └──────────────────┘
```

### 2.2 数据流程
1. **数据收集** → 查询各模块数据库，获取最近N天数据
2. **脱敏处理** → 移除敏感信息，泛化个人数据
3. **格式化** → 转换为AI友好的自然语言描述
4. **AI分析** → 调用DeepSeek API生成洞察
5. **结果解析** → 提取结构化洞察和建议
6. **展示交付** → 在移动端界面显示分析结果

## 3. 核心模块实现

### 3.1 DeepSeekClient (`src/ai/deepseek_client.py`)
- **功能**: 实现与DeepSeek Chat Completion API的交互
- **核心特性**:
  - API密钥管理和安全传输
  - 请求数据构建与格式化
  - 数据脱敏处理（财务数据取整、域名泛化等）
  - 错误处理和重试机制
  - 性能统计监控

- **关键方法**:
  - `analyze_recent_data()` - 统一分析接口
  - `_build_analysis_request()` - 构建标准化请求
  - `_anonymize_*_data()`系列方法 - 各模块数据脱敏

### 3.2 DataCollector (`src/ai/data_collector.py`)
- **功能**: 从各模块收集和预处理数据
- **支持模块**:
  - 财务数据: 查询`financial.db`，获取支出摘要
  - 链接数据: 查询`links.db`或`state.db`，获取分类统计
  - 日历数据: 查询`calendar.db`，获取事件分布
  - 健康数据: 查询`health.db`，获取健康指标

- **数据摘要结构**:
  ```python
  {
      "time_range": {"days": 7, "start_time": 1772384383, "end_time": 1772989183},
      "modules_available": {"financial": True, "links": True, ...},
      "data": {
          "financial": {...},  # 总支出、日均、Top类别等
          "links": {...},      # 链接数、类别分布、域名等
          "calendar": {...},   # 事件数、忙碌时段等
          "health": {...}      # 步数、心率、睡眠等
      }
  }
  ```

### 3.3 AnalysisService (`src/ai/analysis_service.py`)
- **功能**: 整合数据收集和AI分析，提供完整服务
- **核心特性**:
  - 数据收集协调
  - AI请求构建和调用
  - 结果缓存（可配置时长）
  - 结构化洞察提取
  - 性能监控统计

- **主要接口**:
  ```python
  service.analyze_recent_activity(days=7)  # 分析最近活动
  service.get_detailed_insights(result)    # 获取详细洞察
  service.generate_analysis_report(result) # 生成分析报告
  ```

### 3.4 iOS端实现
- **AIAnalysisManager** (`ios/MemoTime/Services/AIAnalysisManager.swift`)
  - Swift原生AI分析管理器
  - 支持DeepSeek和Coze API提供商
  - 数据脱敏和格式化处理

- **AIAnalysisView** (`ios/MemoTime/Views/AIAnalysisView.swift`)
  - SwiftUI分析界面
  - 洞察卡片展示
  - 建议优先级标识
  - 设置配置界面

## 5. 集成指南

### 5.1 后端集成
```python
from ai.analysis_service import AnalysisService

# 创建分析服务
service = AnalysisService(config={
    "api_key": os.getenv("DEEPSEEK_API_KEY"),
    "cache_enabled": True,
    "cache_duration": 3600
})

# 分析最近7天活动
result = service.analyze_recent_activity(days=7)

# 获取详细洞察
insights = service.get_detailed_insights(result)
```

### 5.2 iOS端集成
```swift
import MemoTimeAI

// 使用AI分析管理器
AIAnalysisManager.shared.analyzeRecentActivity(days: 7) { result in
    switch result {
    case .success(let response):
        // 处理分析结果
        print("分析完成: \(response.analysisResults.insights.count)条洞察")
    case .failure(let error):
        // 处理错误
        print("分析失败: \(error.localizedDescription)")
    }
}
```

### 5.3 环境配置
```bash
# 设置API密钥
export DEEPSEEK_API_KEY="your_api_key_here"

# 或使用Coze API
export COZE_API_KEY="your_coze_api_key_here"
```

## 6. 安全与隐私

### 6.1 数据脱敏
- **财务数据**: 金额取整处理（百位数）
- **链接数据**: 域名泛化处理
- **日历数据**: 移除具体事件标题
- **健康数据**: 指标精度处理

### 6.2 传输安全
- HTTPS加密传输
- API密钥不在代码中硬编码
- 请求数据脱敏后发送

### 6.3 本地存储
- 缓存数据不包含敏感信息
- 缓存可配置过期时间
- 支持手动清除缓存
